# 软著设计完整流程 Skill

## 概述

这是一个用于生成软件著作权申请材料的完整流程自动化工具。该 Skill 整合了从需求收集到最终材料整理的全部步骤，帮助用户快速生成符合《计算机软件著作权登记办法》要求的各类申请文档。

## 功能特性

- **自动化流程**：覆盖软著申请的全部5个步骤，从意图确认到最终材料整理
- **规范化输出**：严格按照软著申请的格式要求生成文档
- **模块化设计**：每个步骤可独立使用，也可按流程顺序执行
- **智能生成**：基于用户输入自动生成PRD、使用手册、代码文档等

## 适用场景

当用户提出包含以下关键词的请求时，应调用此 Skill：
- 软著
- 软件著作权
- 软件说明文档
- 软件代码文档
- 代码文档

## 快速开始

### 方式一：完整流程（推荐）

直接告诉 AI 您需要软著设计，系统将自动引导您完成所有步骤：

```
用户：我需要为"智能健康评估系统V1.0"申请软件著作权

AI：[自动进入步骤1] 收集项目信息...
     [完成步骤1] 是否继续生成PRD？
     用户：是
     [完成步骤2] 是否继续生成使用手册？
     用户：是
     [完成步骤3] 是否继续生成代码文档？
     用户：是
     [完成步骤4] 是否继续整理申请材料？
     用户：是
     [完成步骤5] 所有材料已生成完毕！
```

### 方式二：分步骤执行

您可以指定从某个特定步骤开始：

```
# 直接从PRD生成开始
用户：我有prepare.md文件，直接帮我生成PRD

# 直接生成代码文档
用户：帮我生成软著代码文档

# 只整理申请材料
用户：我有所有文档了，帮我整理成申请材料
```

### 方式三：快速生成（自动模式）

使用以下关键词，AI将自动生成所有内容：

```
用户：帮我生成"智能健康评估系统V1.0"的软著材料，我全都要

AI：[自动生成并填充所有必要信息]
```


## 输出文件格式要求

### 申请表
- 字体：宋体8号（所有单元格）
- 源程序量：根据实际代码行数填写
- 编程语言：列出所有使用的语言
- 主要功能：只填写功能描述（第21行）
- 技术特点：单独填写技术架构特点（第22行）

### 使用手册
- 页眉：软件名称+版本号（居中），宋体9号，下边框线
- 页脚：页码（居中），宋体9号
- 页边距：上下2.54cm，左右3.18cm
- 题目：软件全称+版本号（不分页）

### 代码手册
- 每页50行代码（含注释）
- 不得有空行、空白页或重复页
- 字号：等线10号
- 行距：1.04倍（多倍行距）
- 前30页：重点展示特色功能代码
- 后30页：包含收尾逻辑代码
- 禁止提交第三方开源库或框架代码
