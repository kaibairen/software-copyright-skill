## PRD 生成标准
生成的 PRD 必须包含但不限于以下章节，并严格遵循"软著申请文档标准格式"：

```markdown
## 1. 项目概要 (Project Overview)
- 软件全称与版本号。
- 业务背景：描述技术痛点（如传统位点预测的局限性）及本项目的核心价值。
- 目标用户画像。

## 2. 详细功能模块 (Functional Requirements)
- 按"模块名称、功能描述、输入/输出、逻辑流程"的结构拆解。
- 重点描述：数据预处理、LLM 模型推理、结果可视化、用户管理等核心模块。

## 3. 技术架构方案 (Technical Architecture)
- 逻辑架构：描述前端、后端、模型层、数据层的层次关系。
- 核心算法逻辑：描述 Transformer/LLM 如何处理序列数据（需包含数据流向）。
- 技术栈：明确 代码语言、深度学习框架、数据库等具体选型。

## 4. 软著合规性描述 (Copyright Compliance)
- 软件原创性说明：强调 LLM 架构在特定生物领域的应用创新。
- 模块物理划分清单：模拟源代码目录结构，展示模块独立性。
- 关键流程图：使用 Markdown 流程图或文字详述核心算法的闭环。

## 5. 性能与约束 (Non-Functional Requirements)
- 性能指标：响应时间、并发处理能力、模型精度要求。
- 扩展性需求：未来支持更多位点预测的可能性。

# Constraints
- 语言：必须使用简体中文。
- 语气：专业、严谨、客观。
- 深度：拒绝空洞的描述，必须包含具体的 API 接口逻辑、数据格式规范和算法步骤。
- 格式：使用 Markdown 排版，利用标题、列表、表格和代码块增强可读性。
- 规范：必须满足软著的申请要求。
```
