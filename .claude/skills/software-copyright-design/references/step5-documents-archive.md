## 指令
1. **读取软著申请环节文件的全部信息**：读取在"./[项目名-版本号]"目录下生成的所有所有项目对应文件获取信息。

2. **复制软著使用手册模板**：
    - 将在`software-copyright-design`技能下的 `使用手册模板.docx` 复制到 "./[项目名-版本号]/materials"下。

3. **完成代码手册**：
    - 参考"./[项目名-版本号]/code"目录下的readme文件，提取"./[项目名-版本号]/code"目录下所有前端代码生成`frontend-code.md`。
    - 参考"./[项目名-版本号]/code"目录下的readme文件，提取提取"./[项目名-版本号]/code"目录下所有后端代码生成`backend-code.md`。
    - 前后端的代码md文件中都应该只包含代码块和代码块所对应的py文件名。
    - 将在`software-copyright-design`技能下的 `代码手册模板.docx` 复制到 "./[项目名-版本号]/materials"下。

4. **填写申请表信息**：
    - 申请表的具体格式和带填写条目请参考`software-copyright-design`技能下的 `申请表模板.docx`
    - 根据已有信息填充申请表中所有的带填写条目，源程序量和编程语言可以根据"./[项目名-版本号]/code"目录下的readme文件获取
    - 使用python-docx库读取模板文件，保留模板中已有内容（如著作权人、开发者等），只填写空白条目
    - **重要**：填写后必须将所有单元格字体设置为宋体8号

5. **间歇策略**：每个任务可以视作一个子环节，每完成一个子环节询问用户是否进行下一步，采用间歇机制，按顺序逐个完成相关任务

6. **保存**：当对应的内容生成完全时，将整理好的信息保存。
    - 将生成的申请表保存在当前项目的"./[项目名-版本号]/materials"目录下，命名方式为"[软著名称+版本号{V1.0}+文档形式（申请表）].docx"
    - 将生成的软著使用手册保存在当前项目的"./[项目名-版本号]/materials"目录下，命名方式为"[软著名称+版本号{V1.0}+文档形式（使用手册）].docx"
    - 将生成的代码手册保存在当前项目的"./[项目名-版本号]/materials"目录下，命名方式为"[软著名称+版本号{V1.0}+文档形式（代码手册）].docx"

4. **任务完成**：告知用户软著生成流程已经完全结束，最终材料保存在"./[项目名-版本号]/materials"目录下。


## 文件生成要求
### 申请表填写要求
- **填写原则**：
    - 保留模板中已有的内容（如著作权人、开发者、发表状态、权利范围、权利取得方式等）
    - 只填写空白条目
    - **所有条目必须填写**
- **字体要求**（必须满足）：
    - **统一使用宋体8号字体**
    - 在填写完数据后，需要遍历整个表格设置所有单元格的字体
    - 新填写的内容也要单独设置字体
- **源程序量填写**：
    - 根据代码手册的实际行数填写
    - 如：约3000行(前端约1500行,后端约1500行)
- **编程语言填写**：
    - 列出所有使用的语言
    - 如：Python(后端核心), HTML5(前端), CSS3(样式), JavaScript ES6+(交互), SQL(数据库), Jinja2(模板)
- **软件技术特点填写**：
    - 技术特点的描述禁止超过200字要求
- **软件主要功能填写**：
    - 主要功能的描述禁止超过200字要求

- **内容格式**：利用`scripts/generate_application_form.py`生成符合要求的申请表。
